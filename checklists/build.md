# Build checklist — final gate before shipping a built surface

Run this before handing back any screen, component, or copy you produced.

## Before building
- [ ] Answered the Five Questions for this specific screen.
- [ ] Named the single primary action.
- [ ] Derived the product's own tokens (color, type, spacing, radius, motion) — did not impose a style.

## Structure & attention
- [ ] Exactly one primary action, visually singular (Von Restorff), thumb-zone, ≥44×44px/48dp, full-width on mobile.
- [ ] CTA is an outcome statement, not a verb.
- [ ] Visible choices limited and grouped; consequential decisions first.
- [ ] Squint test passes — the primary element dominates.
- [ ] ≤3 live type levels; one dominant anchor; clutter removed.

## Trust, feedback, emotion
- [ ] Outcome demonstrated/previewed before any commitment ask (feedforward).
- [ ] Specific social proof adjacent to the action (if a decision surface).
- [ ] Every action gets ≤200ms feedback; waits show labeled progress, not bare spinners.
- [ ] Error and empty states crafted as carefully as the success state.
- [ ] One peak moment and a genuine ending designed into the flow.
- [ ] Copy in plain language, consistent voice, written for zero-context readers.

## Honesty (hard gate — Module 09)
- [ ] Every default serves the user, not the business at their expense.
- [ ] Every gate/constraint has a real, visible exit (no looping "decline").
- [ ] No fake scarcity, fabricated proof, hidden costs, or confirmshaming.
- [ ] Errors written in the system's voice, not blaming the user.

## Visual language (Module 10)
- [ ] One dominant accent; background on the surface ladder; no off-palette grays.
- [ ] Spacing and radius on their scales; depth via the theme-correct strategy.
- [ ] Canonical components reused; one signature easing; `prefers-reduced-motion` honored.
- [ ] Selection/status conveyed by more than color; focus states visible; AA contrast; aspect ratios locked.

## Handoff
- [ ] Delivered the artifact plus a short rationale mapping each consequential decision to its principle.

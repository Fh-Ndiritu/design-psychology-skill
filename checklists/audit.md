# Audit checklist — final gate before reporting

Run this before delivering any audit. It mirrors the ten modules at a glance.

## Framing
- [ ] Stated the input source and viewport (live URL vs source; mobile vs desktop).
- [ ] Named the page's single primary action.
- [ ] Answered the Five Questions (or flagged any unanswerable).
- [ ] Ran the squint test and recorded what dominates.

## The ten dimensions
- [ ] **Conversion (01):** outcome-led copy; outcome demonstrated before the ask; anchor/decoy/center-stage on pricing; right default; genuine (not fake) urgency; low payment friction.
- [ ] **Attention (02):** one primary action; choices limited (Hick); progressive disclosure; no banner-blind upsells; targets sized/placed by importance; consequential decisions front-loaded.
- [ ] **Visual (03):** ≤3 type levels; one dominant anchor; one highlight (Von Restorff); proximity declares relationships; interactive elements look interactive; same-behavior-same-look; clutter removed; key items at list edges.
- [ ] **Trust (04):** first-50ms polish; consistent spacing/palette/type; promises kept; feedforward CTAs (outcome, not verb); specific social proof at the decision point; ≤200ms feedback; plain-language copy; worst-case-emotional resilience.
- [ ] **Motivation/retention (05):** fast path to Aha; labor illusion on waits; goal-gradient progress; early commitment; accumulated work visible; every gate has an exit; fixed feature locations.
- [ ] **Simplicity (06):** stripped to essentials; ≤5–7 choices; named stages; bounded units; flow state protected; smart defaults, no re-asking.
- [ ] **Emotional (07):** consistent tone; engineered peak + ending; a moment of personality; visible work during waits; immediate reward; narrative framing.
- [ ] **Memory (08):** recognition over recall; visuals over instructional text; error states crafted; recent positive evidence surfaced; change introduced incrementally.
- [ ] **Honesty (09):** restrictions explained; system-voiced errors; honest personalisation; failure measured; assumptions sourced; engagement mechanics checked for gaming. **Any dark-pattern blocklist item → Critical.**
- [ ] **Visual language (10):** tokens consistent; one accent; surface ladder; glow/shadow depth right for theme; signature motion; canonical components; accessibility (targets, focus, contrast, color-independence, reduced-motion).
- [ ] **Meta/SERP (11):** only if SEO surfaces in scope.

## Output quality
- [ ] Every finding has a severity, the violated principle, an exact location, and one concrete fix.
- [ ] Findings ordered by conversion/trust cost, not by module order.
- [ ] Included 2–4 genuine strengths (calibrates the critique).
- [ ] Named the single highest-leverage fix.
- [ ] Scaled the report to the ask (no 30-finding dump for a one-button question).

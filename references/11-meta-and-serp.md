# 11 — Meta Titles & SERP

Governs the `<title>` and meta description for any indexable surface — landing pages, blog posts, programmatic SEO pages, product/location/service pages, event pages. The SERP snippet is the conversion surface *before* the click; treat it with the same rigor as an on-page CTA. Use this module only when SEO surfaces are in scope.

---

## The five core rules

1. **Keyword-first, always.** The exact phrase a person searches leads the title *and* the description. Never lead with the brand.
2. **Fit the window.** Title 50–60 chars. Description 105–155 chars, with the core message in the first ~120 (mobile truncates there).
3. **Earn the click.** Title states a specific value; description leads with the keyword, delivers the value, and **closes with a directional CTA**.
4. **One coherent story.** Title, description, and on-page content say the same thing — this is what stops search engines rewriting your description and gets the page cited in AI Overviews.
5. **Unique per page.** No two pages share a title; no description echoes its own title.

---

## Title rules

- **Length 50–60 chars** (hard floor 50, hard ceiling 60). Under 50 wastes ranking real estate; over 60 truncates mid-word. Emoji + its space counts.
- **Structure:** `primary keyword → value/modifier → (brand, only if a product page)`. Front-load the keyword; engines weight early words.
- **No brand suffix** on editorial/pSEO/event pages — the brand is already in the URL and structured data. Spend every character on keyword + value. Never *lead* with the brand.
- **Title case**, applied consistently. No stop words in the lead position.
- **Proven formulas** (use when they fit, don't force):
  - List — `[Number] [Keyword] [Modifier]` → "25 Small Backyard Ideas for Tiny Yards"
  - How-To — `[Keyword]: How to [Outcome]`
  - Question — the searcher's exact question, verbatim
  - Year — `[Keyword]: [Superlative] [Type] for [Year]` (time-sensitive pages only)
- **Power levers** (only when genuine): numerals beat written numbers; odd numbers win list titles; brackets/parentheses signal a deliverable — `(Step-by-Step)`, `(With Costs)`, `[Free Template]`; authority words ("Complete," "Proven," "Guide") only when earned.
- **Year policy:** add a year *only* where freshness drives the click (pricing, trends, "best tools," annual roundups), and commit to refreshing it annually. Never on evergreen topics — a stale year hurts CTR and trust.

## Description rules

- **Length 105–155 chars.** Core message + keyword + CTA inside the first ~120; use 120–155 for cuttable supporting detail. Never exceed 155.
- **Structure:** lead keyword → value proposition → directional CTA. Contain the exact target phrase early (engines bold the match and rewrite less). Always close with a CTA ("See how," "Compare costs," "Plan yours," or a trailing →).
- **Standalone factual sentence:** AI Overviews reuse the description as citation text without the title — it must make sense alone.
- **Never:** start with the brand; ALL-CAPS or excessive punctuation; keyword stuffing; "Click here…"; a description near-identical to the title; invented stats not supported by the page.

## Emoji (optional, controlled)

If the product runs SERP emoji experiments, treat them as a *controlled* lever, not decoration: one emoji per field max, counted toward the length limit, keyword still leads the text, and only glyphs that actually render in search results (validate render rate). If unsure, ship plain — a clean keyword-first snippet always beats a stripped or broken emoji.

---

## Page-type playbook

- **Blog/informational:** state specifically what the reader learns + signal depth.
- **pSEO (`{dim1} {dim2}`, e.g. city × style):** lead with the exact combination (the query), name a specific qualifier (climate, budget, zone), close with a try-it CTA.
- **Event:** event name + city; what/where + date framing + a "see details" CTA. Build deterministically from structured fields where possible; keep factual; pair with complete Event schema.
- **Location:** city/region + service + a local credibility signal.
- **Service (commercial):** capability + differentiation + next step.
- **Product:** product name + primary benefit + conversion CTA (price/availability if room).

---

## Pre-publish checklist

- [ ] Title 50–60 chars, keyword-first, no brand lead, unique.
- [ ] Title uses a formula and/or a genuine power lever.
- [ ] Year only on time-sensitive pages (and scheduled to refresh).
- [ ] Description 105–155 chars, keyword in the first few words, core message within 120.
- [ ] Description closes with a directional CTA and reads as a standalone factual sentence.
- [ ] Emoji (if used) is controlled, counted, single, and verified to render.
- [ ] No rewrite triggers: brand-first, ALL-CAPS, stuffing, "click here," desc≈title, invented stats.
